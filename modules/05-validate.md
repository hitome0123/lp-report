# Phase 5 — 数据交叉验证

## 目标
在报告生成前，对所有关键数字进行多源交叉验证，确保数据准确性。

## 验证规则
- 每个关键数字至少 **2个独立来源** 验证
- 来源不一致时标注差异，取更权威的来源
- 无法验证的数据标注"中等置信度"，不编造

## 置信度定义

| 等级 | 条件 | 处理 |
|------|------|------|
| **高** | ≥2个权威来源一致 | 直接使用 |
| **中** | 1个来源或推算值 | 使用但标注 |
| **低** | 无可靠来源 | 不使用，标"数据未获取" |

## 验证流程

```
1. 遍历报告中的每个数字
2. 查找对应的搜索结果来源
3. 比对多源数据是否一致
4. 不一致 → 重新搜索补充来源
5. 标注置信度等级
6. 高置信≥80% → 通过，进入渲染
7. 高置信<80% → 补充搜索，重新验证
```
