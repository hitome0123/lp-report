# Phase 3 — 报告文字生成

## 目标
用 GPT-4o 生成7页LP投后汇报报告的全部文字内容，严格LP视角。

## LP写作核心规则

**每段话的自检：**
1. 这段话是在介绍公司，还是在向LP汇报投资？
2. LP读完会知道"我的钱怎么样了"吗？
3. 有没有回答"所以对我们这笔投资意味着什么"？

**视角转换：**
| 写什么 | 错误写法 | 正确写法 |
|-------|---------|---------|
| 行业数据 | "市场从$6B增长到$49B" | "市场爆发为Cerebras提供需求支撑，OpenAI合约已验证切入能力" |
| 融资历程 | "H轮估值$23B" | "Tiger Global领投+AMD入股，IPO确定性提升" |
| 营收数据 | "H1营收$136.4M" | "经营验证：营收超预期，OpenAI合约锁定未来数年营收" |

## 7页内容规格

| 页 | 标题 | 字数 | 核心元素 |
|----|------|------|--------|
| P1 | 封面+组合 | ~420字 | 组合表3行 + 策略说明 |
| P2 | 目录 | ~100字 | 5章链接 |
| P3 | 投资逻辑兑现 | ~1,850字 | 3个thesis卡片 + 高亮框 |
| P4 | 经营数据+行业 | ~980字 | 同行表 + 市场图 + 意义 |
| P5 | 融资+IPO | ~860字 | 融资图 + 技术表 + IPO时间线 |
| P6 | 风险+投后管理 | ~1,200字 | 风险矩阵 + 4卡片 |
| P7 | 结论+速览 | ~1,100字 | 结论框 + 13行数据表 |
| **合计** | | **~6,410字** | |

## 输出
- `report_text.json` — 每页的文字内容（标题、正文、表格数据、高亮框）
- 全文LP视角复审通过后才进入 Phase 4
